## Recursos por categoria

Na referência OpenAPI, os grupos seguem o fluxo natural de integração:
**conta → saldo → receber → enviar → administrar**.

### Conta & saldo

| Categoria      | Endpoints |
| -------------- | --------- |
| **Account**    | Conta da API key (`GET /v1/account`), chaves Pix, limites de pagamento (`GET /v1/account/limits`), tarifas por id (`GET /v1/pix-accounts/{id}/fees`) e política de rate limit (`GET /v1/pix-accounts/{id}/rate-limit`) |
| **Movements**  | Saldo, extrato e consulta de movimentos |

### Receber Pix

| Categoria   | Endpoints                                          |
| ----------- | -------------------------------------------------- |
| **Charges** | Cobranças Pix (criar, listar, consultar, refresh)  |

Pix creditados **sem cobrança** (só pela sua chave Pix) não têm grupo próprio nesta referência: use o webhook **`pix.in.received`** e o extrato **`GET /v1/statement`** / consultas em **Movements** para auditar — fluxo em [Receber um Pix](/guides/receive-pix).

### Enviar Pix

| Categoria   | Endpoints                                  |
| ----------- | ------------------------------------------ |
| **Consulta de chave Pix** | Consultar titular e dados antes da transferência (`/v1/dict/lookup`) |
| **QR Code** | Decodificação de BR Code                   |
| **Transferências Pix** | Envio por chave ou QR Code (`/v1/payouts` …) |

### Administração

| Categoria    | Endpoints                            |
| ------------ | ------------------------------------ |
| **Webhooks** | Cadastro de URLs HTTPS e eventos Pix |

## Headers especiais

| Header                | Quando usar                                                     |
| --------------------- | --------------------------------------------------------------- |
| `Authorization`       | Sempre. `Bearer <api-key>`                                      |
| `Idempotency-Key`     | Em todo POST. 8–64 caracteres                                   |
| `X-Pix-Account-Id`    | Quando a key não está vinculada a uma conta única               |
| `Content-Type`        | `application/json` em requests com body                         |
